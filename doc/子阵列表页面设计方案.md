# 设计方案：系统主机的子阵列表页面

## 一、核心逻辑

### 1. 判断是否显示子阵列表
- 通过 `rpc/getDeviceStatus` 获取 `is_system_master` 字段
- 如果 `is_system_master === true`：显示子阵列表页（多个子阵卡片）
- 如果 `is_system_master === false`：直接显示当前子阵的详情页（原EmsTopology页面）

### 2. 数据获取方式
- 统一使用 `rpc/getDeviceStatus` 主题
- 系统主机会返回所有子阵的汇总信息
- 响应数据结构建议：

```json
{
  "ret": true,
  "cluster": {
    "system_id": 1,
    "cluster_id": 1,
    "is_system_master": true
  },
  "clusters": [
    {
      "cluster_id": 1,
      "is_online": true,
      "master_status": "online",
      "slave_count": 3,
      "online_slave_count": 2,
      "total_power": 125.5,
      "system_soc": 85.2,
      "status": "running",
      "warning_count": 0,
      "fault_count": 0
    },
    {
      "cluster_id": 2,
      "is_online": true,
      "master_status": "online",
      "slave_count": 4,
      "online_slave_count": 4,
      "total_power": 98.3,
      "system_soc": 78.5,
      "status": "running",
      "warning_count": 1,
      "fault_count": 0
    }
  ]
}
```

## 二、子阵卡片设计

每个子阵卡片应该显示以下信息：

### 1. 卡片头部
- **子阵ID**：大号显示 "子阵 1"
- **状态指示灯**：
  - 🟢 绿色：运行中
  - 🟡 黄色：待机/告警
  - 🔴 红色：故障
  - ⚫ 灰色：离线

### 2. 主要数据区（4个关键指标）
```
┌─────────────────────────────┐
│  子阵 1          🟢 运行中   │
├─────────────────────────────┤
│  ⚡ 总功率      125.5 kW    │
│  🔋 系统SOC     85.2 %      │
│  📡 在线从机    2/3         │
│  ⚠️  告警数      0          │
└─────────────────────────────┘
```

### 3. 卡片底部
- **主机状态**：显示该子阵主机的在线状态
- **点击区域**：整个卡片可点击，进入该子阵的详情页

## 四、页面路由结构

```
/ (首页)
├─ 判断 is_system_master
│  ├─ true → 显示子阵列表页 (ClusterList.vue)
│  └─ false → 显示子阵详情页 (EmsTopology.vue)
│
/cluster/:clusterId (子阵详情页)
└─ 显示单个子阵的主从机拓扑 (EmsTopology.vue)
```

## 五、实现步骤

### 1. 修改 useGlobalWebSocket.js
- 在 `systemData` 中添加 `clusters` 数组字段
- 解析 `rpc/getDeviceStatus` 响应中的 `clusters` 数据

### 2. 创建 ClusterList.vue
- 读取 `systemData.cluster.is_system_master`
- 如果为 true，显示子阵卡片列表
- 如果为 false，自动跳转到 `/cluster/1`（当前子阵详情）

### 3. 修改 EmsTopology.vue
- 接收 `clusterId` 参数
- 添加返回按钮（仅在系统主机模式下显示）
- 修改标题显示为 "子阵 X 详情"

### 4. 修改路由配置
```javascript
{
  path: '/',
  name: 'Home',
  component: ClusterList  // 智能判断显示列表或详情
},
{
  path: '/cluster/:clusterId',
  name: 'ClusterDetail',
  component: EmsTopology,
  props: true
}
```

## 六、用户体验流程

### 场景1：系统主机（多子阵）
1. 打开首页 → 看到子阵列表（6个卡片）
2. 点击"子阵2"卡片 → 进入子阵2详情页
3. 看到主从机拓扑图
4. 点击返回按钮 → 回到子阵列表

### 场景2：单子阵（非系统主机）
1. 打开首页 → 直接显示当前子阵详情页
2. 没有返回按钮（因为没有上级页面）
3. 直接操作主从机拓扑

## 七、方案优点

- ✅ 逻辑清晰：通过 `is_system_master` 自动判断
- ✅ 数据统一：只用一个 `rpc/getDeviceStatus` 主题
- ✅ 扩展性好：支持1-8个子阵
- ✅ 用户友好：单子阵时不显示多余的列表页

## 八、技术要点

### 数据流
```
rpc/getDeviceStatus (请求)
    ↓
WebSocket 响应
    ↓
useGlobalWebSocket 解析
    ↓
systemData.cluster.is_system_master
systemData.clusters[]
    ↓
ClusterList.vue 或 EmsTopology.vue
```

### 状态映射
```javascript
const statusMap = {
  'running': { text: '运行中', color: 'green', icon: '🟢' },
  'standby': { text: '待机', color: 'yellow', icon: '🟡' },
  'fault': { text: '故障', color: 'red', icon: '🔴' },
  'offline': { text: '离线', color: 'gray', icon: '⚫' }
}
```

### 卡片数据计算
```javascript
const clusterCards = computed(() => {
  return systemData.value.clusters?.map(cluster => ({
    id: cluster.cluster_id,
    status: cluster.status,
    statusInfo: statusMap[cluster.status],
    power: cluster.total_power?.toFixed(1) || '--',
    soc: cluster.system_soc?.toFixed(1) || '--',
    slaves: `${cluster.online_slave_count}/${cluster.slave_count}`,
    warnings: cluster.warning_count || 0
  })) || []
})
```
